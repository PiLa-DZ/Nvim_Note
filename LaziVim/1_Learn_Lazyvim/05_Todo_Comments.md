```js
// Keyword and Purpose ========================================

// NOTE:  General information

// INFO:

// TODO:  General "to-do" items.
//        Use this for features you haven't finished yet.

// PERF:  Notes about performance issues
//        or areas that need optimization.

// TEST:  Marks code related to testing
//        or places where tests are needed.

// HACK:  For "dirty" code
//        or workarounds that should be cleaned up later.

// WARN:  To warn other developers
//        (or your future self) about something risky.

// BUG:  Highlights a known issue that needs fixing.

// FIX:  Used for code that is broken

// FIXME: Used for code that is broken
//        and needs immediate correction.
//        (They generally do the same thing).
```
